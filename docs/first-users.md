# JarHert — короткий старт

Это личный Hermes-помощник в Telegram. Он сохраняет заметки, ставит
напоминания, ведёт проекты и через подключённый Task Command Center работает с
Trello и Google Calendar.

## Как начать

1. Открой JarHert в Telegram.
2. Напиши обычным текстом, без специальных тегов:
   - `напомни через 30 минут позвонить`;
   - `запиши идею сделать короткий чеклист`;
   - `завтра в 10 проверь сервер`;
   - `покажи план на завтра`;
   - `что у меня сегодня`.
3. Для действий с Trello, Calendar и отправкой сообщений Hermes показывает один
   preview. Нажми `Подтвердить`, если план верный.

## Что можно писать

- Вопрос: `объясни MVP простыми словами`.
- Заметка: `сохрани мысль про утренний digest`.
- Поиск: `найди заметки про OAuth`.
- Напоминание: `напомни завтра утром проверить оплату`.
- Задача: `создай задачу проверить OpenRouter`.
- Календарь: `поставь в календарь демо завтра в 15:30`.
- Голосом: наговори несколько мыслей, Hermes покажет один разбор перед
  применением.

## Подтверждения

Низкорисковые действия выполняются сразу: заметки, напоминания, поиск и чтение
списков.

Среднерисковые действия требуют подтверждения:

- создать задачу;
- переместить задачу;
- закрыть задачу;
- создать событие календаря.

Без кнопки `Подтвердить` такие действия не выполняются.

## Ограничения

- Hermes не читает `.env`, ключи или произвольные файлы сервера.
- Кодовые задачи выполняются только в отдельной sandbox-среде.
- Если Google Calendar OAuth протух, календарные действия не сработают до
  обновления токена.

## Приватность

- Telegram ID используется только для allowlist, ownership и доставки ответов.
- Секреты не печатаются в логах и не хранятся в git.
- События диагностики пишут агрегаты, статусы, trace id и типы действий, но не должны содержать API keys.

## Проверка профиля

```bash
scripts/local_check.sh
```

Для полного внешнего подтверждения владелец явно запускает native gate. Он
пишет в реальный Telegram и создаёт затем удаляет canary в Trello и Calendar:

```bash
NATIVE_RELEASE_ALLOW_LIVE=1 scripts/native_release_gate.sh
```
