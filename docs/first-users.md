# JarHert / AI Brooch — короткий старт

Это Telegram-помощник для личных задач. Он отвечает на вопросы, сохраняет идеи, ставит напоминания, создаёт задачи и события календаря, если интеграции подключены.

## Как начать

1. Открой бот в Telegram и нажми `/start`.
2. Напиши обычным текстом, без специальных тегов:
   - `напомни через 30 минут позвонить`;
   - `запиши идею сделать короткий чеклист`;
   - `завтра в 10 проверь сервер`;
   - `покажи план на завтра`;
   - `что у меня сегодня`.
3. Для действий с Trello/Calendar бот попросит подтверждение. Нажми `Подтвердить`, если всё правильно.

## Что можно писать

- Вопрос: `объясни MVP простыми словами`.
- Идея: `идея сделать digest по утрам`.
- Память: `запомни что токены нельзя печатать`.
- Напоминание: `напомни завтра утром проверить оплату`.
- Задача: `создай задачу проверить OpenRouter`.
- Календарь: `поставь в календарь демо завтра в 15:30`.
- Голосом: скажи ту же фразу, бот расшифрует и выполнит.

## Подтверждения

Низкорисковые действия выполняются сразу: идеи, память, напоминания, список задач.

Среднерисковые действия требуют подтверждения:

- создать задачу;
- переместить задачу;
- закрыть задачу;
- создать событие календаря.

Без кнопки `Подтвердить` такие действия не выполняются.

## Ограничения

- Бот не читает `.env`, ключи, файлы сервера.
- Бот не запускает shell-команды и не управляет сервером напрямую.
- Бесплатные LLM-провайдеры могут отвечать медленно или пусто. Поэтому есть fallback и quality gates.
- Если Google Calendar OAuth протух, календарные действия не сработают до обновления токена.

## Приватность

- Telegram ID используется только для allowlist, ownership и доставки ответов.
- Секреты не печатаются в логах и не хранятся в git.
- События диагностики пишут агрегаты, статусы, trace id и типы действий, но не должны содержать API keys.

## Проверка перед реальным использованием

```bash
scripts/local_check.sh
.venv/bin/python scripts/preflight.py
.venv/bin/python scripts/live_e2e.py \
  --tg-user-id <твой_telegram_id> \
  --send-telegram \
  --require-real-llm \
  --include-task \
  --include-calendar
```

Для голосового сценария сначала создай локальный тестовый файл:

```bash
scripts/create_voice_fixture.sh
.venv/bin/python scripts/live_e2e.py \
  --tg-user-id <твой_telegram_id> \
  --send-telegram \
  --require-real-llm \
  --voice-file dev/voice_fixtures/live_e2e.m4a
```
